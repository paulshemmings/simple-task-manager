About Simple Task Manager


Original Requirement

Recently I found myself wanting a task manager with some specific features. I needed it to allow me to record if the task was blocked by an external resource, and if so what that external resource was. I also wanted to record *how* that blocker was finally resolved. That way I could re-use that information in the future to resolve other blockers. And finally I wanted it to be simple. It had to consist of a single page, with as few clicks as possible required to add/edit a task. The tasks had to be fully searchable, and sortable.

Rather than looking around for an existing solution, I thought it would be more fun just to write one myself. It was certainly quicker.


Technology Stack

AngularJS | Google's Client side MV* framework.
Angular's implementation of two-way data-binding makes CRUD very simple.

Bootstrap | Pre-build Front-end component library.
I'm not a designer - color pallettes mean nothing to me. I didn't want to waste time trying to make the simple table look nice. Bootstrap comes with all the widgets you'd ever need.

Node.JS | Server side Javascript platform
I love Node. It is very light weight, and very extensible. It also means I can write all my code in one language - which is a great plus. I'm not a huge fan of some of the more popular plugins though. So I decided against using Express.

MongoDB | No-SQL document based database server.
This is by far the easiest to use no-sql database server out there. It is very user-friendly. As I would only need to be persisting arrays of single objects, with no interdependencies, this would fit my requirements perfectly.

Mongoose | MongoDB ORM for NodeJS.
A very friendly ORM that plugs into Node with almost no effort. This plug-in makes using MongoDB wonderfully easy.

Jasmine | Testing framework.
The best BDD test framework available.

Karma | Test Runner.
Yet another google product (like AngularJS), this makes running your tests simple.
